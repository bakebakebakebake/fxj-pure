# Cover Suggestions

This file is a lightweight art-direction list for flagship posts on FXJ Wiki. It does not add image assets. The goal is to make future `heroImage` work more intentional and easier to batch produce.

## Recommended Targets

### 1. `interview-operating-system`
- Direction: layered memory map + process/thread state diagram
- Visual language: dark slate background, cyan and amber accents, restrained technical infographic feel
- Why: this article is system-heavy and benefits from a “map of the machine” cover

### 2. `interview-computer-network`
- Direction: protocol stack slices, packets, ports, request/response arrows
- Visual language: deep blue grid, clean line-work, small red highlights for TCP/HTTP state changes
- Why: networking content reads better with a diagrammatic cover than with a generic abstract image

### 3. `interview-java-basic`
- Direction: JVM / class loading / reflection / proxy collage
- Visual language: charcoal + electric blue, card-based layout, information-dense but not noisy
- Why: this is a representative backend fundamentals article

### 4. `solutions-codeforces-1`
- Direction: contest scoreboard fragments, terminal windows, graph paper notes
- Visual language: cool-toned competitive programming atmosphere, subtle red/green status markers
- Why: it signals “contest editorial collection” immediately

### 5. `solutions-atcoder-1`
- Direction: clean Japanese minimalism + algorithm state diagrams
- Visual language: white / navy / mint, a bit lighter and tidier than the Codeforces series
- Why: AtCoder content usually fits a calmer, more geometric look

### 6. `algorithm-game-theory`
- Direction: impartial game tree, winning/losing state marks, chalkboard texture
- Visual language: dark green-black board, thin chalk-like edges, sparse but sharp
- Why: game theory benefits from visual structure rather than decorative photography

### 7. `algorithm-advanced-data-structure`
- Direction: segment tree / Fenwick tree / DS layers as stacked nodes
- Visual language: cool gray base, vivid cyan path highlights, compact node network
- Why: data structure posts need a cover that feels structural and exact

### 8. `improve-concentration`
- Direction: low-noise desk, notebook, soft daylight, single focal object
- Visual language: warm neutral palette, paper texture, quiet and human
- Why: this topic should deliberately break from the “hard technical” visual tone

## General Rules

- Backend / CS basics:
  prefer diagrams, stacks, cards, memory maps, protocol arrows
- Contest solution collections:
  prefer scoreboard fragments, terminal windows, handwritten reasoning scraps
- Long reflective writing:
  prefer quiet still-life scenes, soft light, low contrast, fewer hard edges
- Avoid:
  generic AI neon cities, random code wallpaper, overused purple gradient templates
